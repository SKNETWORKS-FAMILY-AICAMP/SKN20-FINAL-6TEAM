# Task 2 — Bug Fixing

> 목표: 사용자가 실제로 마주칠 수 있는 모든 UI/UX/기능 버그를 식별하고 수정한다.
> 전략: 코드 정적 분석 + /chrome 브라우저 자동화를 통한 실제 테스트 병행

2. 헬스체크 확인:

4. 새 탭 생성하여 `http://localhost` 접속

## Phase 2 — 채팅 기능 테스트 (핵심 기능)

### 정상 플로우
| 시나리오 | 검증 항목 |
|---------|----------|
| 비로그인 사용자 질문 | 게스트 메시지 카운트 증가, 10개 제한 동작 |
| 창업/지원 도메인 질문 | 스트리밍 응답, 출처(Sources) 표시, 도메인 분류 정확도 |
| 재무/세무 도메인 질문 | 동일 검증 |
| 인사/노무 도메인 질문 | 동일 검증 |
| 법률 도메인 질문 | 동일 검증 + 법률 보충 문서 |
| 멀티 도메인 질문 | 복수 도메인 검색 + 통합 응답 |
| 멀티턴 대화 | 이전 대화 컨텍스트 유지, 대명사 해소 |

### Edge Cases
| 시나리오 | 검증 항목 |
|---------|----------|
| 빈 메시지 전송 | 입력 차단 또는 적절한 에러 |
| 특수문자만 입력 | gibberish 감지 → 안내 메시지 |
| 매우 긴 메시지 (2000자+) | 정상 처리 또는 길이 제한 안내 |
| 연속 빠른 전송 | 레이트 리밋 또는 큐잉 동작 |
| 스트리밍 중 페이지 이동 | 스트리밍 중단, 메모리 누수 없음 |
| 스트리밍 중 새 질문 | 이전 스트리밍 취소 후 새 응답 |
| 네트워크 끊김 시뮬레이션 | 에러 메시지 표시, 재시도 가능 |

### /chrome 테스트 스크립트
```
1. navigate → http://localhost
2. 채팅 입력창에 "소규모 창업 지원금 종류가 뭐가 있나요?" 입력
3. 전송 버튼 클릭
4. 스트리밍 응답 완료 대기
5. 출처(Sources) 섹션 존재 확인
6. 액션 버튼(좋아요/싫어요/복사) 동작 확인
```

## Phase 3 — 인증 플로우 테스트

### 정상 플로우
| 시나리오 | 검증 항목 |
|---------|----------|
| 테스트 로그인 | 모달 표시 → 로그인 → 상태 변경 |
| Google OAuth 로그인 | OAuth 플로우 → 콜백 → JWT 발급 |
| 로그아웃 | 토큰 블랙리스트 + 쿠키 제거 + 상태 초기화 |
| 토큰 만료 | 자동 갱신 (refresh token) |
| 게스트 → 로그인 전환 | 게스트 메시지 동기화 (`syncGuestMessages`) |

### Edge Cases
| 시나리오 | 검증 항목 |
|---------|----------|
| 만료된 refresh token | 로그인 페이지로 리다이렉트 |
| 보호 라우트 직접 접근 | `/company`, `/schedule` → 로그인 유도 |
| 관리자 라우트 일반 유저 접근 | `/admin` → 접근 거부 |
| 동시 탭에서 로그아웃 | 다른 탭도 로그아웃 상태 반영 |

## Phase 8 — 크로스 기능 통합 테스트

> 사용자가 여러 기능을 교차 사용할 때 발생할 수 있는 상태 충돌, 메모리 누수, 경쟁 조건을 검증한다.

### 시나리오 1: 채팅 ↔ 페이지 전환
| 시나리오 | 검증 항목 |
|---------|----------|
| 채팅 스트리밍 중 기업 등록 페이지로 이동 | 스트리밍 중단, 메모리 누수 없음, 복귀 시 이전 메시지 보존 |
| 기업 폼 작성 중 사이드바에서 채팅 세션 전환 | 폼 데이터 유실 여부, 경고 표시 |
| 일정 페이지에서 캘린더 조작 중 알림 클릭 | 캘린더 상태 유지, 알림 정상 표시 |

### 시나리오 2: 인증 상태 변경 중 기능 사용
| 시나리오 | 검증 항목 |
|---------|----------|
| 기업 등록 폼 작성 중 토큰 만료 | 자동 갱신 후 폼 데이터 유지, 또는 로그인 유도 |
| 관리자 대시보드 조회 중 로그아웃 | 보호 라우트에서 즉시 리다이렉트 |
| 게스트 채팅 10개 후 로그인 → 즉시 기업 등록 | 게스트 메시지 동기화 + 기업 등록 동시 정상 동작 |

### 시나리오 3: 동시 비동기 작업
| 시나리오 | 검증 항목 |
|---------|----------|
| 채팅 질문 전송 직후 다른 세션으로 전환 | 응답이 올바른 세션에 귀속 |
| 문서 생성 요청 중 새 채팅 질문 전송 | 두 요청 모두 정상 완료, 상태 혼선 없음 |
| 일정 저장 + 기업 수정 동시 제출 | 두 API 요청 독립 처리 |
| 관리자 히스토리 필터 변경 중 새로고침 | 필터 상태 복원 또는 초기화 |

### 시나리오 4: 빠른 연속 조작
| 시나리오 | 검증 항목 |
|---------|----------|
| 사이드바 세션 빠르게 3회 연속 전환 | 마지막 세션만 로드, 깜빡임 없음 |
| 채팅 전송 버튼 더블클릭 | 중복 메시지 방지 |
| 뒤로가기/앞으로가기 빠르게 반복 | 라우팅 상태 정합성 |

### /chrome 테스트 스크립트 (통합)
```
1. 로그인 → 채팅 질문 전송
2. 스트리밍 응답 중 /company 페이지로 이동
3. 기업 등록 폼 일부 작성
4. 사이드바에서 채팅으로 복귀
5. 이전 채팅 메시지 보존 확인
6. 새 질문 전송 → 정상 응답 확인
7. /schedule 페이지 이동 → 일정 생성 → 채팅 복귀
```

## Phase 10 — 버그 수정 및 회귀 테스트

### 실행 절차
1. Phase 2~9에서 발견된 버그 목록 작성
2. 심각도별 우선순위 정렬 (Critical > High > Medium > Low)
3. 각 버그 수정 후 관련 테스트 실행
4. 전체 테스트 스위트 실행:
   - `.venv/bin/pytest rag/tests/ -v`
   - `.venv/bin/pytest backend/tests/ -v`
   - `cd frontend && npm run test`
5. /chrome으로 수정된 버그 재검증
